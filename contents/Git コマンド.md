## 1. ローカルリポジトリの作成・クローン

* **`git init`**：現在のディレクトリに新規ローカルリポジトリを作成
* **`git clone <URL>`**：リモートリポジトリをローカルに複製

## 2. 変更の記録・確認

* **`git status`**：変更されたファイルやステージングの状態を確認
* **`git diff`**：ファイルの変更内容（差分）の確認
* **`git add <ファイル名>`**：指定したファイルをステージングエリア（コミット対象）に追加
* **`git add .`**：すべての変更・新規ファイルを一括でステージングエリアに追加
* **`git commit -m "<メッセージ>"`**：ステージングされた変更をメッセージ付きで記録（コミット）

## 3. 履歴の確認

* **`git log`**：コミット履歴の一覧を表示
* **`git log --oneline`**：コミット履歴を1行ずつ簡潔に表示

## 4. ブランチの操作

* **`git branch`**：ブランチの一覧表示
* **`git branch <ブランチ名>`**：新しいブランチの作成
* **`git checkout <ブランチ名>`**（または `git switch <ブランチ名>`）：指定したブランチへの切り替え
* **`git checkout -b <ブランチ名>`**（または `git switch -c <ブランチ名>`）：新ブランチの作成と切り替えを同時に実行
* **`git merge <ブランチ名>`**：現在のブランチに指定したブランチの変更を取り込み
* **`git branch -d <ブランチ名>`**：不要になったブランチの削除

## 5. リモートリポジトリとの連携

* **`git remote add origin <URL>`**：ローカルにリモートリポジトリ（origin）を登録
* **`git push origin <ブランチ名>`**：ローカルのコミットをリモートリポジトリに送信
* **`git fetch`**：リモートリポジトリの最新情報をローカルに取得（統合はしない）
* **`git pull`**：リモートリポジトリの最新情報を取得し、現在のブランチに自動マージ

## 6. 変更の取り消し・一時退避

* **`git restore <ファイル名>`**：ステージング前のファイルの変更を元に戻す
* **`git reset --hard <コミットID>`**：指定したコミット時点まで完全に状態を巻き戻す
* **`git stash`**：作業中の変更を一時的に退避（別ブランチへの切り替え時に重宝）
* **`git stash pop`**：退避していた変更を現在のブランチに復元
