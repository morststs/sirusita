## 正規表現

文字列のパターンを表現・検索・置換するための記法（Regular Expression）。テキスト処理・入力チェック・ログ解析などで幅広く使われる、メタ文字を中心としたクイックリファレンス。

---

## 1. 文字クラス（Character Classes）

1文字にマッチする種類を指定する記法。`[]` で独自の集合を、`\d` などで定義済みのクラスを表す。

| パターン | 意味 |
| --- | --- |
| `.` | 改行以外の任意の1文字 |
| `\d` | 数字（`[0-9]`） |
| `\D` | 数字以外（`[^0-9]`） |
| `\w` | 単語構成文字（`[A-Za-z0-9_]`） |
| `\W` | 単語構成文字以外 |
| `\s` | 空白文字（スペース・タブ・改行など） |
| `\S` | 空白以外 |
| `[abc]` | a, b, c のいずれか1文字 |
| `[a-z]` | a〜z の範囲の1文字 |
| `[^abc]` | a, b, c 以外の1文字（否定） |

```regex
[A-Za-z]\d
```

```text
マッチ: A1, b9, Z0
非マッチ: 12, AA
```

## 2. 量指定子（Quantifiers）

直前のパターンの繰り返し回数を指定する。`?` を付けると最短マッチ（非貪欲）になる。

| パターン | 意味 |
| --- | --- |
| `*` | 0回以上の繰り返し |
| `+` | 1回以上の繰り返し |
| `?` | 0回または1回（任意） |
| `{n}` | ちょうど n 回 |
| `{n,}` | n 回以上 |
| `{n,m}` | n 回以上 m 回以下 |
| `*?` `+?` `??` | 最短マッチ（非貪欲） |

```regex
a{2,4}
```

```text
マッチ: aa, aaa, aaaa
非マッチ: a, aaaaa（先頭4文字のみマッチ）
```

貪欲と非貪欲の違い:

```text
対象文字列: <p>あ</p>
<.+>   → <p>あ</p> 全体（貪欲）
<.+?>  → <p> のみ（非貪欲）
```

---

## 3. アンカーと境界（Anchors）

文字ではなく「位置」にマッチする。文字列の先頭・末尾や単語の区切りを指定する。

| パターン | 意味 |
| --- | --- |
| `^` | 行の先頭 |
| `$` | 行の末尾 |
| `\b` | 単語境界（単語と非単語の境目） |
| `\B` | 単語境界以外 |
| `\A` | 文字列全体の先頭 |
| `\z` | 文字列全体の末尾 |

```regex
^\d+$
```

```text
マッチ: 12345（行全体が数字のみ）
非マッチ: 12a45, a123
```

```regex
\bcat\b
```

```text
マッチ: a cat sat の cat
非マッチ: category の cat
```

---

## 4. グループとキャプチャ（Groups & Capturing）

`()` でまとめた部分は後で参照・置換に使える。`(?:)` は参照不要のグループ化、`(?<name>)` は名前付き。

| パターン | 意味 |
| --- | --- |
| `(abc)` | グループ化＋キャプチャ（番号付き） |
| `(?:abc)` | グループ化のみ（キャプチャしない） |
| `(?<name>abc)` | 名前付きキャプチャ |
| `\1` `\2` | キャプチャした内容の後方参照 |
| `\k<name>` | 名前付きキャプチャの後方参照 |

```regex
(\d{4})-(\d{2})-(\d{2})
```

```text
対象: 2026-06-28
グループ1: 2026 / グループ2: 06 / グループ3: 28
置換例: $1/$2/$3 → 2026/06/28
```

名前付きキャプチャ:

```regex
(?<year>\d{4})-(?<month>\d{2})
```

---

## 5. 選択と後方参照（Alternation & Backreference）

`|` でいずれかにマッチ。後方参照は同じ文字列の繰り返しを検出するのに便利。

| パターン | 意味 |
| --- | --- |
| `a|b` | a または b |
| `(cat|dog)` | cat または dog |
| `\1` | 1番目のキャプチャと同じ文字列 |

```regex
^(https?|ftp)://
```

```text
マッチ: http://, https://, ftp://
```

後方参照で重複語を検出:

```regex
\b(\w+)\s+\1\b
```

```text
マッチ: the the, is is（同じ単語の連続）
```

タグの開閉一致:

```regex
<(\w+)>.*?</\1>
```

```text
マッチ: <b>太字</b>, <span>x</span>
```

## 6. 先読み・後読み（Lookaround）

マッチ位置の前後を「消費せずに」条件として確認する。実際の結果には含まれない（ゼロ幅）。

| パターン | 名称 | 意味 |
| --- | --- | --- |
| `(?=...)` | 肯定先読み | 直後が ... である |
| `(?!...)` | 否定先読み | 直後が ... でない |
| `(?<=...)` | 肯定後読み | 直前が ... である |
| `(?<!...)` | 否定後読み | 直前が ... でない |

```regex
\d+(?=円)
```

```text
対象: 1500円
マッチ: 1500（「円」は結果に含まれない）
```

```regex
(?<=\$)\d+
```

```text
対象: $100
マッチ: 100（「$」は結果に含まれない）
```

パスワード強度チェック（英大文字・小文字・数字を各1つ以上含む8文字以上）:

```regex
^(?=.*[a-z])(?=.*[A-Z])(?=.*\d).{8,}$
```

## 7. エスケープと特殊文字（Escaping）

メタ文字そのものを検索したいときは `\`（バックスラッシュ）でエスケープする。

| 文字 | エスケープ | 補足 |
| --- | --- | --- |
| `.` | `\.` | ドット記号そのもの |
| `*` `+` `?` | `\*` `\+` `\?` | 量指定子の記号 |
| `(` `)` `[` `]` `{` `}` | `\(` `\)` `\[` `\]` `\{` `\}` | 括弧類 |
| `\|` `^` `$` | `\|` `\^` `\$` | 選択・アンカー記号 |
| `\` | `\\` | バックスラッシュ自身 |
| タブ・改行 | `\t` `\n` `\r` | 制御文字 |

```regex
\d+\.\d+
```

```text
マッチ: 3.14, 12.5（小数点はエスケープが必要）
非マッチ: 314（ドットがない）
```

> 補足: `[]` の中ではほとんどのメタ文字が文字どおりに扱われるため、エスケープは最小限で済む（例: `[.+*]` はそのままドット・プラス・アスタリスク）。

## 8. よく使う実用パターン（Practical Patterns）

実際の入力チェックや抽出でそのまま使えるパターン集。要件に応じて調整して利用する。

| 用途 | パターン |
| --- | --- |
| メールアドレス | `[\w.+-]+@[\w-]+\.[\w.-]+` |
| URL | `https?://[\w./?=&%#-]+` |
| 日付（YYYY-MM-DD） | `\d{4}-\d{2}-\d{2}` |
| 時刻（HH:MM） | `([01]\d|2[0-3]):[0-5]\d` |
| IPv4アドレス | `(\d{1,3}\.){3}\d{1,3}` |
| 郵便番号（日本） | `\d{3}-\d{4}` |
| 電話番号（日本） | `0\d{1,4}-\d{1,4}-\d{4}` |
| 16進カラーコード | `#[0-9A-Fa-f]{6}` |
| 半角数字のみ | `^\d+$` |
| 全角ひらがな | `[ぁ-ん]+` |

メールアドレスの例:

```regex
^[\w.+-]+@[\w-]+\.[\w.-]+$
```

```text
マッチ: user@example.com, info.tech+news@mail.co.jp
非マッチ: user@、@example.com、user example.com
```

IPv4（各オクテット 0〜255 まで厳密に判定）:

```regex
^((25[0-5]|2[0-4]\d|1?\d?\d)\.){3}(25[0-5]|2[0-4]\d|1?\d?\d)$
```

```text
マッチ: 192.168.0.1, 255.255.255.0
非マッチ: 256.1.1.1, 192.168.1
```

> 注意: 正規表現の細かな仕様（後読みの可否、`\d` が全角数字を含むか等）は言語・エンジン（PCRE / JavaScript / Python / Go の RE2 など）により異なる。Go の `regexp` は RE2 ベースで後読みや後方参照に非対応な点に注意。
